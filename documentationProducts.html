Introduction

This doc try to explain the installation and Use of opencart "Pop up Login" extension:

This mod have a part in vqmod so don’t overwrite any opencart core file, i don’t imagine a reason to do but if you want to make the changes directly to your opencart core file can read our guide How to apply vqmod changes directly to our opencart core files

This mod use Jquery, opencarts default installation come with this js library but you have to ensure you have not remove in your shop personalization.

Mod/Extension Content

This mod/extension has the next files:

  • \install\vqmod\xml\popuplogin.xml
  • \install\admin\controller\module\loginpopup.php
  • \install\admin\language\english\module\loginpopup.php
  • \install\admin\view\template\module\loginpopup.tpl
  • \install\admin\view\javascript\jquery\jquery.qtip-1.0.0-rc3.min.js
  • \install\catalog\controller\module\loginpopup.php
  • \install\catalog\language\english\module\loginpopup.php
  • \install\catalog\view\theme\default\template\module\loginpopup.tpl*¹
  • \install\catalog\view\theme\default\stylesheet\module\popuplogin.css*¹
  • \install\catalog\view\theme\default\image\user.png and user-wait.gif*¹

*¹ If you don't use default theme you have to adapt the file path.

Installation

Step 1: Upload to Server
Copy the contents of the "Install" folder to your store's top level directory, preserving the directory structure. This will not overwrite anything as they are all new files.
Step 2: Install Module Into Administration
Login to your store's admin and, from the menu, select "Extensions > Modules", then click the "Install" link in the "Action" column for the "Pop up Login" module. You will see a message with the installation confirm.

Module Configuration

Once we have the module installed we can enter in the configuration page, login to your store's admin and, from the menu, select "Extensions > Modules", then click de "Edit" link in same row of the module. You will see something like this:

This screen let you configure the options of the module, now we’ll continue describing the different option:

Title:
Is the title of the pop up window, the text appears at the top of the window (Point 1 of the image). This module is multilingual so you can set a title for each language.
Validation by Ajax:
This is a new feature that makes the whole login process from the popup by ajax AJAX, read section "Ajax Feature" for more info.
Style Class:
The pop up comes configured with default opencart classes, but this field let you add a custom css class.
Back Shadow:
If you choose "Yes" will be displayed a layer behind the pop up that do not allow any operation in the web until the customer close the pop up or try to logon into the web.(See point 4 of the image)
Open in header link:
If you choose option "Yes" the link of "login" in the header instead of redirecting to the login screen will open the logon popup.
Draggable:
If you choose option "Yes" let the user move the pop up around the screen.
Resizable:
If you choose option "Yes" let the user change the size of the pop up.
Auto show:
You can choose between three options:
  • Always: when the user enters the main page of your web, pop up is displayed (if the user is not logged in) if the user closes the popup and reload main page the pop up reappears.(Only when is not logged in)
  • Never: pop up does not automatically appear never.
  • One time: the pop up is displayed the first time the user access main page, if user close the pop up will not return to appear during this navigation (This functionality use a session cookie, read cookie section for more information).
Status:
Enables and disables the module with this option.

How it works?

Once we have the module configured according to our preferences, we can prove in our store, the auto-show only acts on the *main page of the store and the pop up only will appear when user is NOT logged in. The rest of behaviour depends on the settings you want, here are some interesting examples:

  1. Show only first time: If you set the auto-show to "One time" if the user enters to main page the pop up will be shown, and during this session the user will never see anymore. With session we refer until the user closes their browser (Explorer, Mozilla ...)
  2. No auto show, but show when you click the header link: If you set the auto-show to "Never" and you activate the header link ("Open in header link" option) the pop up not will be shown auto, but when the user click on the header link instead of redirecting to the login screen will open the logon popup.
  3. Always show: If you set the auto-show to "Always" the pop up always will appear when customer go to the main page, only when user is not logged in.
  4. Resizable example: As shown in the image below the user can resize the popup
  5. Ejemplo de Dragable: As the image below shows can move the pop around the screen.

*If you want the "auto show" to work on more sites on your store please contact us, is possible to configure.

Ajax Feature

This new feature make possible to do all login process in the pop up dialog using ajax, this is the behaviour:

  • If the customer incorrectly enter the data is not redirected to login page, the error is shown in the dialog popup
  • If the customer correctly enter the data the popup is closed (automatically) and the page is refresh to actualizate the header and user data.
  • To show the user that the request is being processed the login button has an icon that spins while waiting for the response to the request.

You can easy change the icon and the gif off the login button just replace the images user.png and user-wait.gif in the "catalog\view\theme\default\image" folder.

Changing the style of pop up

As described in the configuration section, you can customize the look of our pop up assigning a own CSS class in the field "class", but there is an alternative more interesting, thanks to that pop up use jquery you can totally change the look with the jQuery UI ThemeRoller, the ThemeRoller is a jquery tool that let you configure the aspect of jquery widgets very easily, see what we should do to change the look of our popup.

Step 1:
We go to the web page themeroller, In the left we can see the themeroller tool and in the right the examples of different jquery widgets, in our case we must look the "dialog" that is the plug-in that uses our mod "pop up login".
Step 2:
In ThemeRoller tool can configure every aspect of our theme in the first tab "Roll Your Own", or on the second tab "Gallery" we can see a gallery of predefined themes. Let us see the predefined themes.
Step 3:
If you click on the image of the "widgets theme" the colour will change and we can see how they apply to our dialog. We choose for example "Redmon" that fits perfectly into our default opencart store.
Step 4:
Once we choose our theme we click download, save the zip file to your computer. This file is the entire jQuery library, we are only interested in the css file that is in the folder css/redmon/jquery-ui-1.9.0.custom.css, we will copy this file to our store on the path \catalog\view\javascript\jquery\ui\themes create a folder "redmon" and will paste css file inside.
Step 5:
Now we only need to change the reference to jQuery css in the header, we go to the file header.tpl in catalog\view\theme\default\template\common and look for the line where says "jquery/ui/themes/" and replace this path for our file path.

And we have prepared our pop up!

Use of Cookies

The Functionality to display only once the pop up use cookies, according to new European regulations you must give all the cookie information to the user, if your store should respect this law that is the information of the cookie that is installed with this mod:

Cookie name Cookie provider Expires after Description of cookie
popuplogin empty session Don’t stores information, only serves to indicate that the user already entered on the web and not have to show the popup again

The cookie will only be in the client's browser during the session, that is, when you close the browser, the cookie disappears and next time you enter the store the popup reappears.
